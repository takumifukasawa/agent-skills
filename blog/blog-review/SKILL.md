# blog-review — はてなブログ記事レビュー (Markdown)

Markdown で書かれたはてなブログ記事を入力に、**内容・日本語・はてな固有**の3面でレビューする。著者は技術ブロガー（CG/WebGL 等）想定。**直す→また見てほしい**の往復（再レビュー）を前提に設計してある。

## 入力

- レビュー対象の Markdown ファイルのパス（引数）。例: `/blog-review entry.md`。
- パスが無ければ「どの md を見ますか？」と確認。複数渡されたら1記事ずつ。

## レビューの3面

### 1. 内容
- **構成・論理** — 話の順序、見出し構造、結論までの流れ、跨躇・欠落・重複。
- **わかりやすさ・読者目線** — 前提説明の不足、専門用語の丁寧さ、想定読者レベルに合っているか。
- **技術的正確さ** — コード/用語/主張の正しさ。**断定はしない**。怪しい記述は「要検証」と印を付け、何を確認すべきかを示す（CG/WebGL/シェーダ等は特に）。
- **タイトル・導入・SEO** — タイトルの引き、冒頭フックで読者をつかめているか、見出しの検索性。

### 2. 日本語（誤字脱字 → 文体改善まで）
- 誤字脱字・誤変換・抜け。
- 表記ゆれ、ですます/である調の混在、半角全角の不統一、送り仮名。
- 読みやすさ・文体: 冗長表現、一文が長すぎる箇所、受け身の多用、語尾（〜です/〜ます）の単調な連続。**提案ベース**で、著者の文体は尊重する。

### 3. はてな固有（Markdown モード前提）
- **目次** — 長い記事なら `[:contents]` の有無を確認。
- **見出し階層** — 本文見出しは `##` から始める（`#` は記事タイトル扱い）。階層の飛ばし(`##`→`####`)がないか。
- **脚注・引用・リンク** — 脚注記法、引用ブロック、リンクは標準 Markdown `[text](url)`。ブログカード埋め込みは URL を単独行に置く運用。
- **画像・アイキャッチ** — alt の有無、アイキャッチ（OGP）に使えそうな画像があるか。
- **カテゴリ/タグ・概要** — 設定漏れがないか（md 本文外の項目は「確認を」と添える）。
- 注意: 記法は **Markdown モード / はてな記法モード**で挙動が変わる。標準 Markdown を基準に見つつ、固有要素（目次/脚注/ブログカード）で不確実な点は「エディタのモードを確認」と添える。断定しない。

## 出力フォーマット（初回レビュー）

3面ごとにグループ化し、各指摘に **ID と優先度**を振る:

```
## 内容
- [C1｜高] 箇所:「<見出し名 or 原文引用>」
  指摘: …
  → 修正案: …

## 日本語
- [J1｜中] 箇所:「…」 / 「誤」→「正」
  …

## はてな固有
- [H1｜低] …

## 総評
- 良い点 / 全体所感
- 優先対応トップ3: C1, J3, C4 …
```

- ID 規則: 内容=C / 日本語=J / はてな=H。優先度=高・中・低。
- 箇所は**見出し名か原文引用**で示す（行番号は編集でずれるため引用優先）。
- 日本語の機械的修正（誤字脱字・表記ゆれ・ですます統一）は「適用候補」としてまとめておく。

## 修正の適用

- ユーザーの承認を得たら、**機械的・校正レベルの修正（J カテゴリ）だけ** md を直接編集して適用する。
- 内容・文体（C / 編集判断が要るもの）は**提案どまり**。著者が選ぶ。文体・トーンは丸ごと書き換えない。
- 適用したら「どの指摘を反映したか」を一覧で返す。

## 再レビュー（往復ループ）★これが肝

直した版で再度呼ばれたら、全文を出し直さず**差分で返す**:

1. **変更箇所の特定** — ファイルが git 管理なら `git diff`（前回レビュー時点との差分）で変わった節に注目。未管理なら全文を読み直す。
2. **前回指摘の消し込み** — 会話中の前回指摘リストを参照し、各 ID を **✅解決 / ⚠️未対応 / 🔁部分対応** で判定（根拠付き）。
3. **新規・デグレ検出** — 編集で混入しやすい誤字や、文の流れが崩れた箇所を拾う。
4. **デルタ出力** — 「解決した指摘 / 残課題 / 新規」を簡潔に。更新版の優先トップ3で締める。

これで「レビュー → 直す → 再レビュー」を軽く回せる。

## ルール

- 技術的主張は「要検証」を基本とし、確証があるものだけ誤りと断じる。
- 推測の修正は「自信度: 低」と明示。確認できないものを正解扱いしない。
- 好みの押し付けをしない。指摘には必ず理由を添える。
- 一度に大量に出して圧倒しない。優先度の高いものから。低優先は件数だけ示してもよい。
